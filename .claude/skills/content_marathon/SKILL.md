# Content Marathon — 14-Day Orchestrator

Orchestrate 28 content items across 14 days with scheduled daily triggers.

## State

- Calendar: `~/hairglasses-studio/docs/strategy/launch-content/content-calendar.json`
- Progress: tracked in calendar JSON (status field per item)
- Schedule: `/schedule list` shows active triggers

## Default loop

Parse `$ARGUMENTS`:

### `start`

1. Write the full 28-item `content-calendar.json` (see Calendar Template below)
2. Create scheduled triggers for autonomous daily execution:
   - Morning: `0 9 * * *` → runs `/content-marathon run`
   - Evening: `0 17 * * *` → runs `/content-marathon run`
   Use `/schedule` skill to create cloud-based RemoteTrigger (survives restarts, no 7-day expiry).
   Fallback: CronCreate with `durable: true` if RemoteTrigger unavailable (requires midpoint renewal at Day 7).
3. Process today's morning item immediately (don't wait for first trigger)
4. Report: marathon dates, total items, schedule times, first item processed

### `run`

Called by the scheduled trigger (or manually). This is the core execution loop.

1. Read `content-calendar.json`
2. Determine current slot: morning (before 1pm local) or evening (after 1pm)
3. Find the next unposted item for today + current slot
4. If no items for today: check for overdue items from previous days, process the oldest one
5. If item found:
   - For R (ready) items: read source, verify numbers, format for platform, commit
   - For E (edit) items: spawn an Agent to read research source and draft the post, commit
   - For W (write) items: spawn an Agent to write from resume + repo code, commit
6. Update item status to "drafted" in `content-calendar.json`
7. Commit calendar state
8. Display the formatted content for manual posting
9. If all 28 items are "drafted" or "posted": send completion notification, cancel triggers

### `status`

Delegate to `/content-queue status`. Show: posted/drafted/ready/overdue counts, today's items, days remaining.

### `stop`

1. List active triggers via `/schedule list`
2. Cancel the morning and evening content-marathon triggers
3. Report: items completed, items remaining, final stats

### `catchup`

Process all overdue items (date < today, status = ready) in chronological order. Useful after vacation or missed days.

1. Find all overdue items
2. Process each one via the `run` logic
3. Report how many were caught up
