# Security Audit

Run a structured pre-publish security review. Covers static analysis, supply-chain integrity, spec-to-code compliance, and a second-opinion checklist. Dry-run safe — no mutations unless the user requests fixes.

## When to use

- Before open-sourcing any repo in hairglasses-studio
- Before cutting a public release or publishing to pkg.go.dev
- When adding a new external dependency
- After pulling in a major upstream update
- When a user says "security review", "scan for vulns", "supply chain check", or "safe to publish?"

## Invocation

`$ARGUMENTS` may be:
- Empty — audit the current repo
- A repo name or path — audit that specific repo
- `--fix` — auto-apply low-risk remediations and report what changed
- `--go`, `--python`, `--node` — run only the matching language track
- `--sast-only`, `--supply-chain-only`, `--compliance-only` — run a single phase

## Phase 1 — SAST (Static Analysis)

Run all SAST tools in parallel. Skip tools that are not installed; report missing tools as ADVISORY at the end.

### Go — gosec

```bash
# Install if missing
which gosec || go install github.com/securego/gosec/v2/cmd/gosec@latest

# Run against repo root (exclude test files by default)
gosec -fmt=json -out=/tmp/gosec-report.json ./... 2>/dev/null || true
gosec -fmt=text ./... 2>&1 | tail -30
```

Severity mapping:
- HIGH confidence + HIGH severity → CRITICAL
- HIGH severity, MEDIUM+ confidence → WARNING
- MEDIUM/LOW → ADVISORY

Common findings to escalate:
- G101: Hardcoded credentials
- G304: File path injection
- G401/G501: Weak crypto (MD5, SHA1)
- G501: Import of unsafe package
- G601: Slice/array implicit memory aliasing

### Python — bandit

```bash
# Run if Python files present
find . -name "*.py" -not -path "*/vendor/*" -not -path "*/.git/*" | head -1 | grep -q . && \
  bandit -r . -x ./vendor,./node_modules -f json -o /tmp/bandit-report.json 2>/dev/null || true
bandit -r . -x ./vendor,./node_modules -ll 2>&1 | tail -40
```

### Multi-language — semgrep

```bash
# Run if semgrep is available
which semgrep && semgrep --config=auto --json -o /tmp/semgrep-report.json \
  --exclude='vendor/' --exclude='node_modules/' --exclude='*.sum' . 2>/dev/null || true
semgrep --config=auto --exclude='vendor/' --exclude='node_modules/' . 2>&1 | grep -E "CRITICAL|HIGH|WARNING|error" | head -40
```

Key semgrep rulesets for this org:
- `p/golang` — Go-specific patterns
- `p/secrets` — Credential/token detection (critical for pre-publish)
- `p/owasp-top-ten` — OWASP Top 10 coverage

### Secrets scan (always run)

```bash
# Check for accidentally committed secrets — run even without semgrep
grep -rn \
  "AKIA[0-9A-Z]{16}\|ghp_[A-Za-z0-9]{36}\|sk-[A-Za-z0-9]{48}\|op://\|password\s*=\s*['\"][^'\"]\|secret\s*=\s*['\"][^'\"]\|api_key\s*=\s*['\"][^'\"]\|token\s*=\s*['\"][^'\"]" \
  --include="*.go" --include="*.py" --include="*.sh" --include="*.env" \
  --include="*.json" --include="*.yaml" --include="*.toml" --include="*.conf" \
  -l . 2>/dev/null \
  | grep -v ".git/\|vendor/\|go.sum\|go.mod\|node_modules/" | head -20
```

Any credential pattern hits are **CRITICAL** — block publish until resolved.

Also check git history for accidental secret commits:
```bash
git log --all --full-history --oneline -- '*.env' '*.key' '*.pem' 2>/dev/null | head -10
git log --all -p --follow -S 'AKIA\|ghp_\|sk-ant\|sk-\|op://' 2>/dev/null | grep "^+.*\(AKIA\|ghp_\|sk-\)" | head -10
```

## Phase 2 — Supply-Chain Audit

### Go — module integrity

```bash
# Verify all modules match their checksums in go.sum
go mod verify 2>&1

# Check for known vulnerabilities in dependencies
which govulncheck || go install golang.org/x/vuln/cmd/govulncheck@latest
govulncheck ./... 2>&1 | head -60

# Detect direct deps that are forks or private mirrors (potential typosquatting)
go mod graph | awk '{print $1"\n"$2}' | sort -u | grep -v "^github.com/hairglasses" | head -40
```

Escalate any govulncheck findings with `CRITICAL` or `HIGH` severity.

### npm / Node.js

```bash
# Run if package.json present
[ -f package.json ] && npm audit --audit-level=moderate 2>&1 | tail -30
[ -f package-lock.json ] && npm audit fix --dry-run 2>&1 | tail -10
```

### Python — pip-audit

```bash
# Run if requirements.txt or pyproject.toml present
( [ -f requirements.txt ] || [ -f pyproject.toml ] ) && \
  (which pip-audit || pip install pip-audit --quiet) && \
  pip-audit 2>&1 | tail -30
```

### License compliance

```bash
# Check for copyleft licenses that conflict with MIT release
which go-licenses || go install github.com/google/go-licenses@latest
go-licenses check ./... 2>&1 | grep -v "^$" | head -30
```

Flag any GPL/LGPL/AGPL dependency as a **WARNING** — must be reviewed before open-source release.

## Phase 3 — Spec-to-Code Compliance

Verify that AGENTS.md, README, and `.well-known/mcp.json` claims match the actual codebase.

### Tool count verification (MCP repos)

```bash
# Count registered MCP tool handlers vs documented count
ACTUAL=$(grep -c 'TypedHandler\[\|AddTool\|tool\.New' $(find . -name "*.go" -not -path "*/vendor/*") 2>/dev/null | awk -F: '{sum+=$2} END{print sum}')
CLAIMED=$(grep -oP '(?<=\*\*)(\d+)(?= tools)' README.md 2>/dev/null | head -1)
echo "Actual tools: $ACTUAL | README claims: $CLAIMED"
[ -f .well-known/mcp.json ] && MANIFEST=$(python3 -c "import json,sys; d=json.load(open('.well-known/mcp.json')); print(len(d.get('tools',[])))" 2>/dev/null) && echo "Manifest tools: $MANIFEST"
```

If actual count diverges from README claim by more than 5%, flag as **STALE_DOCS**.

### Module/package count

```bash
ACTUAL_PKG=$(find . -name "*.go" -not -path "*/vendor/*" | xargs dirname 2>/dev/null | sort -u | wc -l)
CLAIMED_PKG=$(grep -oP '(?<=\*\*)(\d+)(?= package)' README.md 2>/dev/null | head -1)
echo "Actual packages: $ACTUAL_PKG | README claims: $CLAIMED_PKG"
```

### AGENTS.md instruction drift

Check that documented build commands in AGENTS.md actually work:

```bash
# Verify the canonical build commands from AGENTS.md exist and run
BUILD_CMD=$(grep -A2 "Build\|build" AGENTS.md 2>/dev/null | grep "^\`\`\`\|^  " | grep "go build\|make build\|npm build" | head -1 | tr -d '`')
[ -n "$BUILD_CMD" ] && echo "Testing: $BUILD_CMD" && eval "$BUILD_CMD" 2>&1 | tail -5
```

### .gitignore completeness

```bash
# Verify sensitive file patterns are in .gitignore
for pattern in ".env" "*.key" "*.pem" "*.p12" "*.pfx" "credentials.json" ".env.local"; do
  grep -qF "$pattern" .gitignore 2>/dev/null || echo "MISSING .gitignore entry: $pattern"
done
```

### Personal info grep (pre-publish critical)

```bash
# Scan for personal paths, emails, or identifiers per sensitivity rules
grep -rn \
  "/home/hg\|hairglasses\.studio\|mitch@\|mixellburk\|monitor.*serial\|employer" \
  --include="*.go" --include="*.py" --include="*.sh" --include="*.md" \
  --include="*.json" --include="*.yaml" --include="*.toml" \
  . 2>/dev/null \
  | grep -v ".git/\|vendor/\|go.sum\|AGENTS.md\|CLAUDE.md\|GEMINI.md\|*.local.md" | head -20
```

Any personal path or email hit in non-instruction files is **CRITICAL** for public repos.

## Phase 4 — Second-Opinion Security Review

After automated scans, apply this adversarial checklist manually (or flag for human review):

### Authentication & Authorization
- [ ] No hardcoded credentials, tokens, or secrets anywhere in the codebase
- [ ] API keys sourced from environment variables or 1Password (`op read`)
- [ ] No auth bypass patterns (empty password checks, commented-out auth)
- [ ] File path inputs validated against traversal (`../`, absolute paths)

### Injection & Execution
- [ ] Shell commands constructed with `exec.Command` (Go) not `sh -c` with user input
- [ ] SQL queries use parameterized statements, not string concatenation
- [ ] No `eval()` or `exec()` with untrusted input (Python/JS)
- [ ] `#nosec` annotations in Go are justified with a comment explaining why

### Data Handling
- [ ] Temporary files use `os.CreateTemp` / `mktemp`, not predictable names in `/tmp`
- [ ] Sensitive data not logged (passwords, tokens, PII in log statements)
- [ ] Cryptography uses standard library primitives, not homebrew crypto
- [ ] Race conditions checked: run `go test -race ./...` if not already done

### Dependencies
- [ ] No pinned-commit dependencies that could be silently swapped
- [ ] No `replace` directives in `go.mod` pointing to local or forked modules (unless intended)
- [ ] All fork dependencies are intentional and documented in AGENTS.md

### Disclosure
- [ ] `SECURITY.md` or security policy present if repo is public
- [ ] No internal hostnames, IP ranges, or org topology exposed in code or comments
- [ ] Makefile targets don't expose internal endpoints or CI secrets in output

## Output format

```
## CRITICAL (N items) — Block publish until resolved
1. [type] — [file:line] — [description] — [remediation]

## WARNING (N items) — Fix before release
1. [type] — [file:line] — [description] — [remediation]

## ADVISORY (N items) — Low risk, investigate when convenient
1. [type] — [description] — [context]

## STALE_DOCS (N items) — Documentation doesn't match code
1. [claim] — [expected vs actual] — [file to update]

## CLEAN — Checks that passed
- SAST: no HIGH/CRITICAL findings
- Supply chain: all modules verified
- Secrets: no credential patterns
- .gitignore: all sensitive patterns covered
- Compliance: tool/package counts match README

## Tools not installed (install for full coverage)
- [tool] — [install command]

## Second-opinion checklist status
- [x] Item confirmed clean
- [ ] Item needs manual review: [reason]
```

## Remediation guidance

After reporting, offer to:
1. Run `go mod tidy && go mod verify` to fix supply-chain issues
2. Add missing `.gitignore` entries
3. Rotate any exposed secrets (generate replacement with `op` or `openssl rand -hex 32`)
4. Create `SECURITY.md` with responsible disclosure template
5. Add `govulncheck` to CI pipeline (`.github/workflows/`)

Commit fixes in phases: supply-chain fixes first, then SAST remediations, then doc updates. Never combine credential rotation with functional changes.
