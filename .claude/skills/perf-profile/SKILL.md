# perf_profile — Wayland / NVIDIA frame-time + shader cost measurement

The stack runs at 240Hz on an RTX card. Every visual knob (blur, shadow, VFR,
VRR, screen shader, DarkWindow per-window shader) is measurable via compositor
overlay, nvtop, or mangohud. This skill is the structured workflow for:

- Measuring idle GPU load
- A/B testing a single setting reversibly
- Choosing a shader tier for a given scenario
- Deciding when to flip `quality` ↔ `performance` mode

## The measurement stack

| Layer | Tool | What it measures |
|---|---|---|
| Compositor | `hyprctl debug:overlay 1` | Hyprland FPS + frame time per monitor |
| GPU | `nvtop` | NVIDIA engine utilization, VRAM, power, clocks |
| App (Vulkan/GL) | `mangohud <app>` | Per-app frame time, FPS, CPU/GPU load |
| Frame-timing | `gamescope -W W -H H --expose-wayland -- app` | Microsecond frame timing via mangoapp |
| Damage | `hyprctl debug:damage_tracking` + visual inspection | Which regions redraw |

## The workflow

### 1. Baseline

```bash
# Snapshot current state
hyprctl monitors -j | jq '.[] | {name, refreshRate, vrr}'
hypr-perf-mode.sh status

# Kick off nvtop in another terminal (ticker-based, readable)
nvtop &
# or mangohud for the target app
MANGOHUD=1 mangohud firefox
```

Note the baseline idle 3D engine % in nvtop. Healthy desktop on RTX at 240Hz
with full quality mode: 20-40% idle. After perf-mode: <15%.

### 2. Enable the compositor overlay

```bash
hyprctl keyword debug:overlay 1
```

Hyprland paints its own FPS + frametime overlay per monitor. On the 240Hz DP-3
you should see frame times hovering <4.2ms. On the 180Hz DP-2 you should see
<5.5ms.

### 3. Change one thing

```bash
# Example: flip VRR to fullscreen-only
hyprctl keyword misc:vrr 2

# Example: reduce blur
hyprctl keyword decoration:blur:passes 1
hyprctl keyword decoration:blur:size 4

# Example: toggle VFR (careful — breaks DarkWindow shader animation)
hyprctl keyword misc:vfr true
```

### 4. Observe

- Watch nvtop 3D engine %
- Watch Hyprland overlay frametime
- Move a window, drag the cursor, scroll in firefox — induce load
- Check for visual regressions (tearing, flicker, cursor corruption)

### 5. Decide

- If better: persist to `hyprland/hyprland.conf` or `hypr-perf-mode.sh`
- If worse or artifacts: revert via `hyprctl keyword <key> <old-value>` or `hyprctl reload`

### 6. Clean up

```bash
hyprctl keyword debug:overlay 0
```

## Dedicated MCP tools

Four purpose-built MCP tools wrap this workflow:

- `mcp__dotfiles__hypr_perf_mode` — get/set perf mode (quality|performance|auto|status)
- `mcp__dotfiles__hypr_vrr_status` — per-monitor VRR state + refresh rate snapshot
- `mcp__dotfiles__hypr_frame_overlay` — toggle compositor FPS overlay
- `mcp__dotfiles__shader_tier` — list shaders by perf tier (cheap/mid/heavy), with optional regenerate

Example invocations:

```
mcp__dotfiles__hypr_perf_mode mode=status         # read current state
mcp__dotfiles__hypr_perf_mode mode=auto           # toggle quality↔performance
mcp__dotfiles__hypr_vrr_status                    # JSON per-monitor VRR
mcp__dotfiles__hypr_frame_overlay enable=true     # show overlay
mcp__dotfiles__shader_tier tier=cheap             # list ambient-safe shaders
mcp__dotfiles__shader_tier regenerate=true        # rebuild all tier playlists
```

## Performance-mode toggle

`scripts/hypr-perf-mode.sh` wraps a curated set of quality/performance deltas:

| Setting | quality | performance |
|---|---|---|
| `decoration:blur:passes` | 2 | 1 |
| `decoration:blur:size` | 6 | 4 |
| `decoration:blur:vibrancy` | 0.22 | 0.15 |
| `decoration:shadow:enabled` | true | false |
| `animations:enabled` | true | true |
| `misc:vfr` | false | true |
| `debug:overlay` | false | true |

Keybind: `$mod CTRL ALT Q` toggles between modes.

VFR tension: `performance` mode sets `vfr=true`, which freezes DarkWindow
animated shaders (time-based effects stop moving when no damage). This is
intentional — the point of perf mode is to drop GPU load; animated shaders
are a luxury that costs ~5-15% idle GPU at 240Hz.

## Shader cost measurement

For the 139-shader DarkWindow catalog, use `shader_benchmark` MCP tool:

```bash
# Via MCP
mcp__dotfiles__shader_benchmark shader=neon-glow
# Returns: {"compile_ms": X, "file_size": Y, "lines": Z}

# Planned extension (Phase 3): runtime GPU delta
# Applies shader via darkwindow:shadeactive, reads nvtop 3D % delta over 5s window
```

Shader tiers (to be generated during Phase 3 benchmarking):

- **tier-cheap** (`<1% GPU, <0.5ms`) — safe for ambient rotation at 240Hz
- **tier-mid** (`1–5% GPU, 0.5–2ms`) — good for focused windows
- **tier-heavy** (`>5% GPU, >2ms`) — reserved for intentional aesthetic showcases

## NVIDIA-specific failure modes

| Symptom | Likely cause | Fix |
|---|---|---|
| Idle GPU >60% at rest | Blur too aggressive at 240Hz | `hypr-perf-mode.sh performance` |
| Cursor tearing / disappearing | Hardware cursor enabled on NVIDIA | `hyprctl keyword cursor:no_hardware_cursors true` |
| VRR brightness flicker | VRR always-on on Samsung VA panel | `hyprctl keyword misc:vrr 2` (fullscreen-only) |
| Driver freeze at idle | Driver 560 explicit-sync KMS regression | `hyprctl keyword render:explicit_sync_kms 0` |
| Ironbar/discord corruption | DMA-BUF modifier mismatch | Add `env = AQ_NO_MODIFIERS,1` in hyprland.conf |
| Fullscreen games stutter | Direct scanout disabled | `hyprctl keyword render:direct_scanout 1` |
| Animated shaders stutter | VFR on + DarkWindow | `hyprctl keyword misc:vfr false` |
